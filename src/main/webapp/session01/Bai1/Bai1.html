Sơ đồ

Chức năng của từng thành phần:

1. Client:

Gửi yêu cầu HTTP đến máy chủ và hiển thị giao diện người dùng dựa trên HTML/CSS/JS.

2. Web Server:

Tiếp nhận yêu cầu HTTP từ client

Phân phối các tài nguyên tĩnh

Chuyển tiếp yêu cầu động đến Application Server.

3. Application Server:

Xử lý logic nghiệp vụ.

Triển khai các thành phần Java như Servlet, Controller, Service, DAO.

Giao tiếp với Database để lấy hoặc cập nhật dữ liệu.

4. Database

Lưu trữ dữ liệu của ứng dụng như thông tin người dùng, sản phẩm, đơn hàng...

Cung cấp dữ liệu thông qua các truy vấn SQL.

Cách các thành phần này tương tác với nhau.

Khi người dùng gửi yêu cầu từ trình duyệt, yêu cầu HTTP được gửi đến Web Server (như Apache Tomcat). Nếu yêu cầu là động (ví dụ: cần xử lý dữ liệu), Web Server sẽ chuyển nó đến Application Server. Tại đây, các thành phần như Servlet hoặc Controller xử lý logic nghiệp vụ, có thể truy cập cơ sở dữ liệu để lấy hoặc cập nhật thông tin. Dữ liệu từ Database được gửi lại cho Application Server, sau đó trả kết quả về Web Server để phản hồi cho client dưới dạng HTML hoặc JSON. Toàn bộ quá trình này tạo nên một ứng dụng web Java đầy đủ chức năng và có thể mở rộng.
